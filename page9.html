Mobile

Mobile technology is getting better and better, so much so that the current mobile phones are capable of high speed internet access. This allows you to access any webpage and watch as many videos as you want.

But not all is good with mobile technology, for it comes with small screen that users use to view most of the web content. Meaning Developers have to optimize their website to fit in the small screen of the mobile phones.

Using web pages on a mobile phone involve these few gestures associated with navigation of you site

  • Swiping up and down
  • Zooming in and out
  • Pinching in and out

These gestures improved the way users access the site and made it clear that mobile technology would dominate the web market.

With all this in mind mobile technology and how users access the web on their mobile device will change in the next years to come.

 

It’s all about the tradeoffs

It’s important to realize, that with small screens there will always be tradeoffs. Because whenever you make any design you are constantly dealing with constraints which cause the tradeoffs.

Most usability problems are caused by poor decisions on what needs to be tradeoffs.

 

Design notes to consider when making a web page for mobile

  • Allow zooming: This is an alternative to making a responsive site which allows a use to zoom into the mobile web page in hopes of seeing the content clearly.
  • Use correctly implemented links that guide the user to the correct parts of the webpage.
  • You could use affordances to illustrate to the user that the area is clickable. Affordances are visual cues that add 3-dimensional style to some parts of your web page. For affordances to work they need to be visible.
  • Note because mobile devices don’t come with a curse, this means that the hover effect will not work so features that use the hover feature will not work either. Like tool tips.
  • You might consider using a flatter design when creating some parts of you web page for they will have less visual noise.
  • Some users are used to very fast speeds while accessing content on the mobile device. It’s important to ensure that your site does not take long to load content because not all mobile phones have the capability to access high speed internet.

 

Mobile apps usability attributes

The attributes included in the definitions of usability are

  • Useful
  • Learnable
  • Memorable
  • Effective
  • Efficient
  • Desirable
  • Delightful

But the most important things to consider are whether the site is

  • Delightful
  • Learnable

 

Making apps delightful

This simply combining ideas of apps that people would enjoy being able to do. For example, you can combine listening to a song, scrolling and sync music together to create a really good user experience

Making apps learnable

This can be done by minimizing the number of feature that are offered by the mobile web page or app.

For example, creating a web app that does one thing, create a to do list.

Making apps memorable

Once a user has figured out how to use an app, they must be able to remember how to use the app next time they want to.

This is done by making the functionality of the web app clear so that the use will not spend a lot of time learning it or need to relearn it when he next tries to use it.
