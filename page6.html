Designing navigation.

If a user cannot figure out how the web page is designed, there is a high chance they will not use you site for a long time.

To give a user good opportunity to navigate you site it’s important to have hierarchy that the user can follow to get what he or she wants.

This hierarchy is placed at the top of your webpage and can be followed to lead you to any part of the webpage.

Web navigation

  • It is as a result of trying to find something and you can either ask by searching on the search bar.
  • You could browse through the web page hoping to find what you looking for.
  • Remember if the use can’t at this stage find what they are looking for they will leave the website.

The lightness of browsing

It’s important to associate the content of you web page to resemble real world events or features. Like the words used to describe the content of a webpage should be a reflection of the real world. For example, using words like cruising browsing or surfing.

Things to remember about a webpage while making navigation

  • Web pages have no sense of scale, it’s difficult to tell how big a web site is.
  • Web pages have no sense of direction because there’s no left or right when dealing with webpages.
  • Web page luck a sense of location. They are a source of knowledge about things.

Important consideration when creating navigations

  • Navigation is designed to tell the user what is there by making the hierarchy clear the user can understand what the web contains to that particular point.
  • Navigations give the user an idea on how to use the site. If you implement the navigation of your site correctly, the navigation can serve as instructions to the user.
  • It makes users have confidence in the web developers that created the web site. Confident users will always come back for more.

Web navigation conventions

Navigation conventions allow the user to easily locate them with minimum effort and creating navigation with a standard appearance make it easy to distinguish them from other parts of the website. 

These conventions could include

  • Site ID or logo
  • Utilities
  • Page name
  • Local navigation
  • Footer navigation

Persistent Navigation

This is a term used to describe navigation elements that can be found on each page. This gives the users the feeling that they are still no the same site and has a very big impact on usability.

Persistent Navigation should include 4 elements

  • Site Id
  • Sections
  • Utilities

Note: no pages where the use will be filling in a form persistent navigation can only be a distraction.

Site ID

This is like the logo of the website and is used to indicate to the user whether or not they have left the site.

The site Id should have a distinctive typeface that makes it look unique when compared to other parts of the webpage.

Sections

This is sometimes known as primary navigation and are links to the main section of the webpage. They are the top level to the sites hierarchy.

They might include an area for a subsection or secondary navigation which when clicked could reveal a dropdown menu that contains links to other parts of the webpage.

The Utilities

These are links to important parts of the site which include

  • Registering to the site
  • Finding help about the site
  • Site map
  • Shopping Cart

Utilizes vary from webpage to webpage and could include customer service or contact us.

 

Home Page.

The home page is one of the most important items found in persistent navigation and is a reassurance that no matter how lost you are you can always find your way to the beginning.

Most users expect the site Id to be the home page so you can also include code to allow the site id to take a user back to the home page when it is click.

Search

Most people prefer searching to browsing, this is why it’s important to have a search box at the top left corner of your webpage.

When creating a search bar, it’s important to avoid these few things.

  • Fancy words: don’t replace the word search with any other alternative because search is an internationally recognized word and should work.
  • Instructions: Don’t add any unnecessary instructions to the search bar most users understand how to use it.
  • Options: Only have one options when creating the search bar more than that might create confusion.
