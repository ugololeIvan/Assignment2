Find some things for them to do.

You have to give people tasks that they can perform which will allow you to evaluate how well you have designed your website. The task includes;

  • Testing the thing you want them to try.
  • Expand the take into stages with contain any details of context they’ll need to know to aid them in performing the task.

Coming up with a list of tasks

Start by making a list of items on your website that you think need testing.

You can also list down your most critical task. Make sure to created test stages that a use has to complete in-order to finish the entire test.

These stages can then be evaluated by you to determine whether the site is useable or not.

You can offer what is known as pilot test scenarios. Which involve reading the scenarios to the test group with the aim of making sure that the scenarios are clear and complete.

For example, on this website I tested.

  • User navigation
  • How good the readability is, by asking one of my class to read a few of the written paragraphs.

You could add a few restrictions to the user as you carry out the test.

  • The user shouldn’t use search feature unless you are testing the search feature.
  • The user should stay on the site for the entire period of the testing process so as to maximize the time allocated during testing.
